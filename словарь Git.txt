1)Чтобы авторизироваться вводим команды :
   git config --global user.name "ваше имя в гитхаб"
   git config --global user.email "ваша почта"
2) pwd - выводит путь к текущему рабочему каталогу
3) Чтобы создать новый репозиторий используется комнада:
   git init
4) git commit -m "___" - фиксация изменений и записи их в репозиторий.
5) ls-la - показывает все файлы в директории.
6) git status - сотояние рабочего каталога.
7) Чтобы сохранить изменения используется команда:
   git add .
8) Команда git log - показывает историю коммитов.
9) git push - используется для выгрузки содержимого локального репозитория в удаленный.
10) Команда git pull - извлекает и загружает содержимое из удаленного репозитория в локальный.
11) Чтобы создать доп.ветку:
     git checkout -b branch1
12) gitignore - игнорирование файлов.
13) git merge - выполняет слияние одной или нескольких веток в  текущую.
14) git rm - удаление файлов из индекса и рабочей копии.
15) git branch - "менеджер веток" перечисляет ветки, создает новые, удаляет и переименовывает их.
16) git checkout - позволяет переключиться между ветками.
17) git stash - используется для временного сохранения всех незакоммиченных изменений для очистки рабочей директории без необходимости коммитеть незавершенную работу в новую ветку.
18) git remote - служит для управления списком удаленных репозиториев.
19) git show - отображает объект в простом и понятном виде.
20) git stash pop - выводит скрытые временные изменения.
21) git remote -v - показывает удаленные репозитории
22) git diff - выводит изменения всех файлов.
23) git restore - команда чтобы отменить изменения в рабочем каталоге.
24) git blame <имя> - эта команда позволяет изучить определенные этапы изменений файла и узнать, кто внес последние изменения в файл
25) git restore <имя файла> - отменить изменения в опр.файле
26) Что бы удалить из индекса некоторые файлы используйте :
    git restore --staged <имя файла>  
27) git restore --hard - 
28) git restore --soft <> - 
29) git commit --amend -m "новый коммит" - 
30) git remote add <имя файла> - присваивание имя репозиторию
